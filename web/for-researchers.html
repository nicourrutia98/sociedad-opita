¿Para quién es esta página?

Esta página es para investigadores académicos — antropólogos, sociólogos, politólogos, psicólogos, economistas, etólogos, periodistas de investigación, estudiantes de posgrado — que quieran usar Sociedad Opita como herramienta o fuente de datos para su investigación.

Si eres solo curioso, mejor explora el mapa directamente.

1. Cómo citar Sociedad Opita

Cita del producto (en papers)

Si publicas paper académico basado en Sociedad Opita:

@misc{sociedad_opita_2026,
  author = {{Sociedad Opita Project}},
  title  = {Sociedad Opita: Forensic Simulation of a Rural Colombian Society},
  year   = {2026},
  url    = {https://sociedad.opitacode.com},
  note   = {Forensic simulation of Tello, Huila (Colombia) with 29 documented personas, Big Five, Lomnitz reciprocity, and Dunbar social layers}
}

Cita de un perfil específico (en papers)

Si haces referencia a una persona específica del producto:

@misc{opita_don_rosalio_2026,
  author = {{Sociedad Opita Project}},
  title  = {{Don Rosalío: Perfil psicosocial}},
  year   = {2026},
  howpublished = {Sociedad Opita Dataset, Tello, Huila},
  note   = {Big Five O=30 C=88 E=38 A=52 N=35, Lomnitz A primary, Muletillas: asina es la cosa, le digo yo}
}

Estos BibTeX se pueden generar automáticamente desde el botón Exportar → BibTeX en la herramienta.

Cómo citar en cuerpo del texto

Estilos sugeridos:

  • APA: (Sociedad Opita Project, 2026)
  • MLA: (Sociedad Opita Project)
  • Chicago: (Sociedad Opita Project 2026)

2. Usos legítimos en investigación

Sí recomendado

  • Análisis secundario: usar los datos para análisis cuantitativos propios (red, regresiones, clustering si crece n).
  • Comparación cualitativa: usar las biografías como material de comparación con trabajo de campo propio.
  • Estudio de simulación: usar el producto como caso de estudio sobre simulación social con LLM (sesgo, representación, etc.).
  • Pedagogía: usar el producto en clases de sociología, antropología, o ciencia de datos para mostrar conceptos.
  • Replicación: replicar la metodología en otra ciudad o contexto rural.

Con cuidado

  • Estudio de un solo perfil: si tu paper depende fuertemente de UNA persona, mejor cita el trabajo de campo original (cuando exista) o usa el producto como referencia secundaria.
  • Análisis estadístico: con n=29 los análisis son direccionales. Reconoce la limitación explícitamente.

NO recomendado

  • Como sustituto de trabajo de campo: el producto NO es etnografía. Cualquier investigación seria sobre Tello requiere trabajo de campo propio.
  • Para inferencia clínica o psicológica individual: los Big Five son derivados cualitativos, no mediciones.
  • Para decisiones legales o políticas sobre personas: no usar para atribuir comportamiento real a alguien.
  • Como "datos reales": el producto es simulación con datos derivados. Reconócelo explícitamente.

3. Exportar datos para tu análisis

Desde la herramienta puedes exportar en 6 formatos:

Formato Para qué Compatible con
JSON Datos estructurados completos (grafo + psicometría + biografías) Python, R, JavaScript, cualquier lenguaje
CSV Tabla plana para análisis estadístico R, pandas, SPSS, Stata, Excel
BibTeX Citas para papers académicos LaTeX, BibDesk, Zotero (con Better BibTeX)
Markdown Informe narrativo en texto plano Obsidian, Roam, Notion, cualquier editor
PDF Informe con metodología citada Cualquier visor PDF
Grafo JSON Visualización de red + notas Obsidian, Roam Research, Cytoscape, Gephi

Ejemplo de uso: análisis en Python

import pandas as pd
import networkx as nx
import json

# Cargar export JSON
with open("sociedad-opita-export.json") as f:
    data = json.load(f)

# Convertir a DataFrame
personas = pd.DataFrame(data["city_full_data"]["personas"])
print(personas[["persona_id", "display_name", "age", "archetype"]].head())

# Construir grafo
G = nx.Graph()
for p in data["city_full_data"]["personas"]:
    G.add_node(p["persona_id"], **p["big_five"])
    for ally in p.get("network", {}).get("aliados", []):
        G.add_edge(p["persona_id"], ally)

# Calcular betweenness
bet = nx.betweenness_centrality(G)
print(sorted(bet.items(), key=lambda x: -x[1])[:5])

5. Replicar para tu propia investigación

Si quieres usar el marco metodológico en otro contexto rural (no Tello, no Colombia), la arquitectura es data-driven.

Tres caminos

  1. Replicación directa: usa el código y los YAML como base, cambia los datos por los tuyos.
  2. Replicación con framework propio: usa los principios metodológicos (Big Five derivado, Lomnitz, Dunbar) pero con tu propio protocolo.
  3. Replicación asistida: contacta al autor del proyecto para colaboración.

Cómo empezar

# 1. Clonar el repositorio (próximamente público)
git clone https://github.com/[autor]/sociedad-opita.git
cd sociedad-opita

# 2. Instalar dependencias
pip install -r requirements.txt

# 3. Crear tu propia ciudad (data-only)
mkdir cities/mi_pueblo/personas
cp cities/_template/* cities/mi_pueblo/

# 4. Llenar los YAML con tus datos
# (ver cities/_template/README.md)

# 5. Validar
python -c "from city_factory import load_city; print(len(load_city('mi_pueblo').personas))"

# 6. Correr simulaciones
# (ver docs/deployment/API.md)

Ver cities/_template/README.md para guía completa. Tiempo estimado: 5 horas por ciudad para 15 personas bien documentadas.

7. Notas para revisores de papers

Si revisas un paper que use Sociedad Opita, considera estas preguntas:

  • ¿El autor declara las limitaciones del producto?
  • ¿El autor distingue entre "datos derivados" y "datos medidos"?
  • ¿El autor valida con trabajo de campo propio?
  • ¿El autor cita las fuentes académicas del marco?
  • ¿Las conclusiones son proporcionales a las limitaciones?
  • ¿Hay problemas éticos no abordados?

Si el paper falla alguna de estas preguntas, es razonable pedir revisión. Sociedad Opita es una herramienta, no una fuente de verdad.

Una nota final

Sociedad Opita es un proyecto pequeño. Si tu investigación usa este producto, te pedimos:

  • Ser explícito sobre qué partes son simulación y cuáles son datos derivados.
  • Reconocer las limitaciones en el paper.
  • Citar apropiadamente.
  • Si encuentras errores o problemas, reportarlos al autor.

Con estos mínimos, este proyecto puede ser útil para la comunidad académica sin traicionar sus principios.
